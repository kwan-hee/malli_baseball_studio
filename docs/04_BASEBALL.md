# 04. 야구사전 채널 규칙

야구 용어·규칙 하나당 롱폼 1편. 기존 파이프라인(C:\야구사전)에서 검증된 방식 유지.
기존 `/야구사전` 스킬(baseball-dictionary)과 산출물 형식 동일하게 유지.

## 말리 페르소나 (야구사전 버전 — 동화 채널과 다름, 주의)

- 야구를 20년 본 동네 형/누나 같은 해설 캐릭터. 친근한 존댓말, 청자는 "여러분".
- 어려운 규칙을 일상 비유로 설명 ("인필드 플라이 = 심판이 미리 아웃이라고 선언해주는 보험").
- 말버릇: "자, 여기서 재밌는 게 뭐냐면요", "이거 은근 헷갈리시죠?"
- 가벼운 유머 1~2회, 정보 정확성 우선. 과장 표현("충격","경악") 편당 1회 미만.
- 금지 문체: "살펴보겠습니다", "알아볼게요" 같은 기계 표현.
- TTS: Gemini TTS 남성 보이스 + 해설자 연기 지시문 (2026-07-07 개정 — Edge InJoon은 기계적이라 폴백 강등, docs/07_AUDIO.md)
- 강아지 말리 캐릭터(docs/03_MALLI.md)는 썸네일·삽화 마스코트로만 사용.

## 1편 구성 산출물

1. 리서치 노트 — 용어 기원·역사·KBO 실제 사례 (출처 포함)
2. 말리 페르소나 3~5분 롱폼 대본
3. SEO 패키지 — 제목·설명·태그·썸네일 텍스트

## 대본 구조

1. 훅 (0~15초): 궁금증 유발 질문 또는 KBO 명장면 언급
2. 용어 정의: 쉬운 말로 한 줄 정의 먼저
3. 기원·역사: 유래 스토리텔링
4. KBO 사례: 실제 경기 사례 1~2개
5. 마무리: 요약 + 다음 편 예고 + **구독·좋아요 요청 필수 (상시 규칙 — 2026-07-07 사용자 확정)**
   예: "오늘 내용이 쓸만했다면 구독이랑 좋아요 부탁드려요. 다음 편에서 만나요!"
   해설자 말투로 자연스럽게 — 한 문장으로 짧게.

## 리서치 규칙

- 유래·연도·인물은 반드시 웹 검색으로 교차 확인. 추측 서술 금지.
- KBO 사례는 실제 경기(날짜, 팀, 선수) 특정 가능해야 사용.

## 대본 분량 (검증값)

- 분당 약 300자 기준, 총 1,000~1,500자 (3~5분). "짧게"=900~1,000자, "길게"=1,400~1,500자.
- 시청자 질문 던지기 2~3회 (리텐션 장치). 고인물용 디테일 1개 이상 배치.

## 영상 제작 (기존 검증 파이프라인)

1. 대본 → 씬 ~10개 분할, 씬별 점수판(emotion/motion/importance) 채점
2. 엔진 결정: motion 7~10 그리고 importance 85+ 씬만 영상 생성, 나머지는 정지 삽화 + Ken Burns
   - 영상 생성 플랫폼: **Higgsfield Ultra 우선, 불가 시 Google Flow** (docs/01_COST_POLICY.md)
3. 삽화: Nano Banana / 나레이션: Gemini TTS 남성 보이스 + 해설자 연기 지시문 (쇼츠는 Hedra) — docs/07_AUDIO.md
4. 조립(ffmpeg): 1280x720 30fps, 씬 여백 0.8초, 오디오 딜레이 400ms,
   Ken Burns 홀짝 교대 줌인(1→1.10)/줌아웃(1.10→1)
5. 자막: 문장별 SRT (글자수 비례 타이밍), Malgun Gothic 18pt, Outline 2, MarginV=10 (바닥 밀착 — 상시 지시)
6. BGM: The Diamond.mp3 @ 15%, 루프, 끝 2초 페이드아웃

- 최종본: Obsidian vault 50_Outputs/야구사전/ 단일 파일
