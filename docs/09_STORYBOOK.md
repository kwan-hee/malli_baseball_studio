# 09. 말리 동화 채널 규칙

말리 캐릭터(docs/03_MALLI.md)가 들려주는 창작 동화 채널. 이 문서가 동화 콘텐츠 진실의 원천.

## 타깃·포맷 (2026-07-07 확정)

- 타깃 연령: 5~7세
- 편당 길이: 3~4분
- 어휘 수준: 타깃 연령이 아는 단어만. 어려운 단어 쓰면 말리가 바로 풀어 설명.
- 문장: 짧게. 한 문장 15자 내외. 의성어·의태어 적극 사용.

## 이야기 구조 (표준 5단계)

1. 말리 인사 + 오늘 이야기 소개 (15초 내외 — 편당 3~4분이라 짧게)
2. 도입: 주인공과 배경 소개
3. 전개: 사건 발생 → 시도와 실패
4. 절정·해결: 주인공이 해결
5. 마무리: 교훈 한 줄 + 말리 마무리 인사 ("오늘 이야기 어땠어요?")
   + **구독·좋아요 요청 필수 (상시 규칙 — 2026-07-07 사용자 확정)**
   예: "재미있었다면 구독이랑 좋아요, 꾹 눌러 주세요! 멍멍! 다음에 또 만나요!"
   말리 말투로 귀엽게 — 딱딱한 안내 금지.

## 이야기 규칙

- 한 편에 교훈 하나만. 두 개 섞지 말 것.
- 교훈은 설교하지 않고 이야기로 보여줌. 마지막에 한 줄로만 정리.
- 금지: 폭력, 공포 조장, 죽음 직접 묘사, 특정 종교·정치 소재
- 반복 구문 활용 (아이들이 따라 말할 수 있는 후렴 구조 권장)

## 그림체 (확정)

- 스타일: 지브리풍 일본 애니메이션 + 수채화 질감 (말리 캐릭터 시트와 동일 — docs/03_MALLI.md)
- 기준 프롬프트: prompts/malli_character.txt 에서 파생
- 한 편 안에서 그림체·주인공 외형 절대 혼용 금지
- **지브리 스타일 필수화 (2026-07-12 사용자 지시 — 드리프트 재발 방지 3중 장치)**
  1. STYLE 문구 단일 출처: `prompts/ghibli_style.txt` (에피소드 스크립트가 런타임에 읽음 — 복사본 드리프트 차단)
  2. 스타일 앵커 이미지 첨부 필수: `images/style_ref/ghibli_scene_ref.png` (승인된 지브리 프레임)를
     캐릭터 시트와 함께 매 이미지 생성 요청에 2번째 레퍼런스로 첨부, "match EXACTLY" 지시
  3. 스타일 게이트: gen_images 후 10장 전수 프레임 검수 — 지브리 셀 톤 이탈 시 해당 씬 재생성,
     통과 전 다음 단계(TTS/클립/합성) 진행 금지
  - 앵커 갱신: 더 좋은 지브리 톤 컷이 나오면 ghibli_scene_ref.png 교체 (최신 승인 컷 유지)
- 장면 수: 편당 8~10컷, 컷당 Ken Burns 줌으로 20~30초 유지
- **출력 해상도: 1280x720 (2026-07-15 사용자 지시 — 다음 편부터. 콩쥐팥쥐까지는 1080p 유지).** compose.py `W, H = 1280, 720`. 이미지 원본은 Ken Burns 여유 위해 기존대로 크게 생성.

## 1편 구성 산출물

1. 이야기 개요 (주인공, 교훈, 줄거리 3줄)
2. 전체 대본 (나레이션 호흡 단위 줄바꿈)
3. 장면 분할표 (컷별 이미지 설명 + 대응 나레이션 구간)
4. SEO 패키지 — 제목·설명·태그·썸네일 텍스트
5. **제목 최적화 (자동 실행)** — SEO 뒤에 제목 엔진을 반드시 돌린다. 규칙 = `docs/11_TITLE_ENGINE.md` (또는 title-optimizer 스킬). 산출: SEO/CTR 점수·S~C 등급·초보(부모) 검색 의도·키워드 앞배치·클릭 유도 제목 10~15개·베스트 1개·모바일 길이 최적화. 동화 채널 규칙(동화명 앞배치 + 감성 + 검색태그 전래동화/잠자리동화/유아동화) 적용. 결과를 md SEO 섹션에 `🎯 제목 최적화`로 추가.
